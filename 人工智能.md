## Transformer 与 Diffusion 核心原理 `2026-06-13`

### Transformer

**一句话：** 让模型能"注意到"序列中任意两个位置之间的关系，并行处理整个序列。

- **Self-Attention**：每个词和句中所有词做关联度打分。例："我把苹果吃了，它很甜"——"它"和"苹果"关联分最高
- **Multi-Head Attention**：多组注意力并行，每组关注不同类型的关系（语法、语义、位置）
- **Positional Encoding**：Transformer 本身不知词序，需额外注入位置信息
- **为什么革命性**：RNN/LSTM 逐词串行→无法并行训练→长序列遗忘开头。Transformer 并行处理全部词，训练快一个数量级
- **架构变体**：GPT 只用 Decoder（自回归生成）、BERT 只用 Encoder（双向理解）、Claude/Gemini 等大模型均基于此
- **训练方式**：自回归 = 逐个预测下一个 token。GPT 给"我今天"预测"去"，给"我今天去"预测"超市"。推理时也是这样逐个生成

### Diffusion（扩散模型）

**一句话：** 学习"如何从噪声中还原图像"，从而学会从纯噪声生成新图像。

- **前向加噪**：给真实图逐步加随机噪声（~1000 步），最终变成纯高斯噪声——人为定义，无需学习
- **反向去噪**：训练神经网络从噪声一步步"猜回"原图——这是模型要学的能力
- **为什么叫扩散**：信息像墨水滴入水中逐步"扩散消失"，去噪则反过来从浑浊水中重新聚拢墨水
- **DDPM**（2020）：基础扩散，1000 步逐步去噪，质量好但慢
- **Stable Diffusion**：不在像素空间去噪，在 VAE 压缩的潜空间（latent space）去噪，大幅降计算量——文生图的基础
- **Diffusion Transformer（DiT）**：用 Transformer 替代传统 U-Net 做去噪网络，OmniDreams/Cosmos 用的就是 DiT

### 两者的交汇

| | Transformer | Diffusion |
|---|---|---|
| 核心问题 | 词与词之间什么关系 | 如何从噪声中恢复信号 |
| 数学本质 | 注意力权重矩阵 | 马尔可夫链 + 概率分布变换 |
| 经典场景 | NLP / 序列建模 | 图像 / 视频生成 |
| 交集 | DiT — 用 Transformer 的规模化能力替代 U-Net 做去噪 |

- **DiT 为什么重要**：U-Net 小规模好但扩展性差，DiT 让扩散模型享受 Transformer 的规模化红利（更多参数→更好效果），这是 Cosmos/OmniDreams 能到 2B 参数的基础

<h3 style="color: #059669;">缓存命中率（Prompt Cache Hit Rate）</h3>

<blockquote style="border-left: 4px solid #059669; padding: 12px 16px; background: #ecfdf5; margin: 12px 0; border-radius: 0 4px 4px 0;">
💡 <strong>一句话总结</strong>：模型处理 prompt 时会计算 KV Cache（注意力键值对），重复前缀可复用已有缓存——命中率越高，推理越快越便宜。
</blockquote>

- **KV Cache 原理**：模型为每个 token 计算注意力层的 Key/Value 向量（大量矩阵运算），计算结果会被缓存
- **缓存命中**：新请求的 prompt 前缀与之前请求相同，这部分 token 的 KV 值直接复用，无需重算
- **缓存命中率** = `命中 token 数 / 总 token 数`
- **核心收益**：
  - 💰 **省钱**：缓存命中 token 通常只收正常价格的 **10%~25%**
  - ⚡ **更快**：跳过大量计算，首 token 响应时间（TTFT）显著降低
  - 🖥️ **省资源**：减少 GPU 显存和算力占用
- **提高命中率的技巧**：
  - 把 system prompt、few-shot 示例等**不变内容放在 prompt 最前面**且不要频繁改动
  - 多轮对话保持历史消息不变，只在末尾追加新消息
  - 长文档、参考资料等静态内容放在开头
- **比喻**：厨师已备好 90% 食材（缓存命中），只需现炒最后 10%（新增 token）——命中率越高，上菜越快、成本越低
