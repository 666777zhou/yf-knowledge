## ESP-IDF

</summary>

<h3 style="color: #0891b2;">ESP-IDF 是什么</h3>

<blockquote style="border-left: 4px solid #0891b2; padding: 12px 16px; background: #ecfeff; margin: 12px 0; border-radius: 0 4px 4px 0;">
💡 <strong>一句话总结</strong>：ESP-IDF 是乐鑫 ESP32 系列芯片的官方开发框架，集成了 RTOS、驱动、协议栈、构建系统和交叉编译工具链，是固件开发的"操作系统 + SDK"一体化平台。
</blockquote>

- **全称**：Espressif IoT Development Framework，乐鑫官方为 ESP32 系列芯片提供的固件开发框架
- **核心组成**：
  - 底层驱动：Wi-Fi / BLE、SPI / I2C / UART、GPIO、ADC 等外设驱动
  - 中间件：lwIP（TCP/IP 协议栈）、MbedTLS（加密）、FAT / SPIFFS（文件系统）
  - RTOS：基于 FreeRTOS，提供任务调度、内存管理、中断处理
  - 构建系统：CMake + `idf.py` 命令行工具（配置、编译、烧录、监控）
  - 工具链：交叉编译器（xtensa / riscv）+ OpenOCD 调试器
- **类比**：ESP-IDF 之于 ESP32，类似 Android SDK 之于 Android 手机

---

<h3 style="color: #7c3aed;">ESP-IDF 版本对比</h3>

<blockquote style="border-left: 4px solid #7c3aed; padding: 12px 16px; background: #f5f3ff; margin: 12px 0; border-radius: 0 4px 4px 0;">
💡 <strong>一句话总结</strong>：v5.5.1 是 2025 年 9 月发布的稳定 bugfix 版本（非最新），最新稳定版为 v6.0（2026 年 2 月），v6.0 有重大破坏性变更，初学者用 v5.5.x 更友好。
</blockquote>

| 版本 | 发布日期 | 类型 |
|------|----------|------|
| v5.4 | 2025 年 1 月 | 主要版本 |
| v5.5 | 2025 年 7 月 | 主要版本 |
| **v5.5.1** | **2025 年 9 月 1 日** | Bugfix 版（用户当前使用） |
| v5.5.3 | 2025 年底 | v5.x 最新稳定版 |
| **v6.0** | **2026 年 2 月 27 日** | **最新主要版本** |

<h3 style="color: #dc2626;">v6.0 破坏性变更（从 v5.x 升级需注意）</h3>

<blockquote style="border-left: 4px solid #dc2626; padding: 12px 16px; background: #fef2f2; margin: 12px 0; border-radius: 0 4px 4px 0;">
💡 <strong>一句话总结</strong>：v6.0 移除了大量传统驱动 API、更换了 C 库和加密 API，从 v5.x 升级需要修改代码适配。
</blockquote>

- **Picolibc 替代 Newlib**：C 标准库换为更轻量的 Picolibc，内存占用更小
- **传统驱动移除**：ADC、DAC、I2S、Timer Group、PCNT、MCPWM、RMT、温度传感器等旧版驱动被删除
- **MbedTLS 4.x + PSA Crypto API**：加密 API 从传统 MbedTLS 迁移到 PSA Crypto
- **警告即错误**：编译警告默认视为错误，代码质量要求更严格
- **新 CMake 构建系统（预览）**：支持 CMake Presets
- **部分组件迁移**至 ESP Component Registry（包管理器）

## I2S

</summary>

<h3 style="color: #4f46e5;">I2S 数字音频总线</h3>

<blockquote style="border-left: 4px solid #4f46e5; padding: 12px 16px; background: #eef2ff; margin: 12px 0; border-radius: 0 4px 4px 0;">
💡 <strong>一句话总结</strong>：I2S 是专门为数字音频设计的串行总线，把时钟和数据分开走线以保证精确采样，SPI/UART/I2C 都无法替代它传音频。
</blockquote>

| 信号线 | 全称 | 作用 |
|--------|------|------|
| **SCK** (BCLK) | Serial / Bit Clock | 位时钟，每个脉冲传输 1 bit |
| **WS** (LRCLK) | Word Select | 声道切换，低=左声道，高=右声道 |
| **SD** (DIN/DOUT) | Serial Data | 数据线，按位串行传输 |
| **MCK** (MCLK) | Master Clock | 主时钟（可选），= 采样率 × 位深 × 256 |

- **时序**：WS 翻转一次 = 一帧（一个声道），数据在 BCLK 边沿采样，先传 MSB
- **标准位深**：16 / 24 / 32 bit，常见采样率 `44.1kHz` / `48kHz` / `96kHz`
- **Master 模式**：ESP32 自己产生 SCK/WS（接 DAC 如 MAX98357、PCM5102）
- **Slave 模式**：外部 Codec 提供时钟（接独立音频芯片如 ES8388）

---

<h3 style="color: #0891b2;">ESP32 上 I2S 的特殊用法</h3>

<blockquote style="border-left: 4px solid #0891b2; padding: 12px 16px; background: #ecfeff; margin: 12px 0; border-radius: 0 4px 4px 0;">
💡 <strong>一句话总结</strong>：ESP32 的 I2S 外设被"魔改"成高速并口 DMA 引擎，不仅传音频，还能驱动 LCD 屏幕和摄像头。
</blockquote>

- **标准 I2S 模式**：接 DAC / Codec 播放音频
- **LCD 模式**：驱动并口 LCD 屏幕，利用 I2S 的 DMA 高速发送像素数据
- **Camera 模式**：从摄像头并口读取图像数据
- **PDM 模式**：直接连接 MEMS 麦克风（脉冲密度调制）
- ⚠️ ESP-IDF v6.0 已移除传统 I2S 驱动，需迁移到新驱动 API

## SPI / I2C / UART

</summary>

<h3 style="color: #4f46e5;">三合一速览</h3>

<blockquote style="border-left: 4px solid #4f46e5; padding: 12px 16px; background: #eef2ff; margin: 12px 0; border-radius: 0 4px 4px 0;">
💡 <strong>一句话总结</strong>：SPI、I2C、UART 是嵌入式最基础的三种板上串行通信协议——要快选 SPI、要省引脚选 I2C、要简单/调试选 UART。
</blockquote>

| 特性 | SPI | I2C | UART |
|------|-----|-----|------|
| 线数 | 4（+ 每从一片 CS） | 2（SDA + SCL） | 2（TX + RX） |
| 速度 | 快（50MHz+） | 慢（100k~3.4Mbps） | 中等（通常 115200~4Mbps） |
| 拓扑 | 一主多从 | 一主多从（最多 127 设备） | 一对一 |
| 全双工 | ✅ | ❌（半双工） | ✅ |
| 有时钟线 | ✅（SCLK） | ✅（SCL） | ❌（靠约定波特率） |
| 协议标准 | 无（厂商自定义） | 有（地址 + 寄存器） | 无（仅物理层） |
| 典型场景 | SD 卡、Flash、屏幕 | 传感器、RTC、EEPROM | GPS、调试串口、蓝牙 AT |

---

<h3 style="color: #2563eb;">SPI — 高速王者</h3>

<blockquote style="border-left: 4px solid #2563eb; padding: 12px 16px; background: #f0f4ff; margin: 12px 0; border-radius: 0 4px 4px 0;">
💡 <strong>一句话总结</strong>：四线全双工、主控节奏、片选选设备——最快但线最多，适合机不离板的高速场景。
</blockquote>

- **4 根线**：`MOSI`（主出从入）、`MISO`（主入从出）、`SCLK`（时钟）、`CS`（片选）
- 主设备产生时钟，从设备跟随；**全双工**，同时收发
- 每个从设备独占一根 CS 脚，通信前拉低对应 CS 选中目标
- 没有统一协议层——每个芯片的指令集/寄存器由厂商定义，需查数据手册

---

<h3 style="color: #d97706;">I2C — 省引脚大师</h3>

<blockquote style="border-left: 4px solid #d97706; padding: 12px 16px; background: #fffbeb; margin: 12px 0; border-radius: 0 4px 4px 0;">
💡 <strong>一句话总结</strong>：两根线挂一百多个设备，靠地址点名——最省引脚但最慢，低速传感器首选。
</blockquote>

- **2 根线**：`SDA`（数据）、`SCL`（时钟），所有设备共享同一对总线
- 每个设备有 7bit 唯一地址，主设备先发地址"点名"，再收发数据
- **半双工**，同一时刻只能收或发
- 有标准协议（起始位、地址帧、ACK、数据帧、停止位），无需查寄存器手册即可通信

---

<h3 style="color: #7c3aed;">UART — 最简调试口</h3>

<blockquote style="border-left: 4px solid #7c3aed; padding: 12px 16px; background: #f5f3ff; margin: 12px 0; border-radius: 0 4px 4px 0;">
💡 <strong>一句话总结</strong>：最古老的串行接口，无时钟、靠约定波特率、起始/停止位定界——简单可靠，是嵌入式调试的第一入口。
</blockquote>

- **2 根线**：`TX`（发）→ `RX`（收），交叉连接，**全双工**
- **无时钟线**——双方提前约定波特率（`115200` 最常见），靠各自的内部振荡器采样
- 字节前后加起始位（0）和停止位（1）来界定帧边界
- 只能一对一通信；异步特性意味着双方时钟无需同步，硬件实现最简单
- 是 ESP32 的默认日志/控制台输出口（通过 USB-UART 桥接芯片连电脑）
